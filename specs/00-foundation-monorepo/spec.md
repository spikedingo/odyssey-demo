# Spec 00 — Foundation & Monorepo

## What

Set up the pnpm workspace + Turborepo monorepo skeleton that all other specs build on. This includes the directory structure, package boundaries, root scripts, TypeScript/ESLint/Prettier configuration, Docker-based local Postgres, and environment variable conventions.

## Why

Everything else depends on a clean, consistent foundation. Getting the workspace boundaries, dependency direction, and toolchain right at the start prevents structural debt in every subsequent spec. The required tech stack (pnpm workspace + Turborepo) is non-negotiable per the assignment.

## Package Boundaries

| Directory | Package name | Role | May depend on |
|---|---|---|---|
| `packages/shared` | `@odyssey/shared` | Pure utility functions (formatters, currency helpers, date utils) | Nothing internal |
| `packages/types` | `@odyssey/types` | Shared enums and non-generated TypeScript types | Nothing internal |
| `packages/ui` | `@odyssey/ui` | React Native + Web UI primitives | `@odyssey/shared` |
| `packages/api-client` | `@odyssey/api-client` | Generated API client + React Query hooks | `@odyssey/types` |
| `services/backend` | `backend` | Hono API server | `@odyssey/shared`, `@odyssey/types` |
| `apps/dashboard` | `dashboard` | Expo RN Web dashboard | `@odyssey/api-client`, `@odyssey/ui`, `@odyssey/shared`, `@odyssey/types` |

No circular dependencies. No `apps/*` → `services/*` imports. No `services/*` → `apps/*` or `packages/api-client` imports.

> **Assignment alignment**: The assignment lists `packages/shared`, `packages/types`, `packages/api-client`. This project adds **`packages/ui`** as a dedicated shared design-system package (implements the assignment's "shared packages for UI/utilities/types" requirement). Utilities remain in `packages/shared`; UI primitives live in `packages/ui`.

## Root Scripts

Defined in root `package.json` (delegated via Turborepo):

```jsonc
{
  "scripts": {
    "dev:dashboard": "turbo run dev --filter=dashboard",
    "dev:backend": "turbo run dev --filter=backend",
    "gen:contract": "turbo run build --filter=backend && pnpm --filter=api-client generate",
    "lint": "turbo run lint",
    "typecheck": "turbo run typecheck",
    "test": "turbo run test",
    "db:up": "docker compose up -d postgres",
    "db:down": "docker compose down",
    "db:migrate": "turbo run db:migrate --filter=backend",
    "seed": "turbo run seed --filter=backend"
  }
}
```

## Database: Local Postgres via Docker Compose

`docker-compose.yml`:
```yaml
services:
  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: odyssey
      POSTGRES_USER: odyssey
      POSTGRES_PASSWORD: odyssey
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U odyssey"]
      interval: 5s
      timeout: 3s
      retries: 5

volumes:
  postgres_data:
```

`DATABASE_URL` for local dev: `postgresql://odyssey:odyssey@localhost:5432/odyssey`

### Switching to Neon (cloud Postgres)

To use Neon instead of local Docker:
1. Create a Neon project at [neon.tech](https://neon.tech).
2. Copy the connection string from the Neon dashboard.
3. Update `services/backend/.env`: `DATABASE_URL=<neon-connection-string>?sslmode=require`
4. No code changes required — Drizzle uses `DATABASE_URL` directly.
5. Skip running `pnpm db:up`; Neon is always-on.

## Acceptance Criteria

- [ ] `pnpm install` succeeds from workspace root with no peer dependency errors.
- [ ] `pnpm dev:dashboard` starts the Expo dev server.
- [ ] `pnpm dev:backend` starts the Hono/Wrangler dev server on port 8787.
- [ ] `pnpm db:up` starts Postgres; `pnpm db:migrate` applies all migrations without error.
- [ ] `pnpm seed` populates the database; running it twice is idempotent.
- [ ] `pnpm lint` passes with zero errors across all packages.
- [ ] `pnpm typecheck` passes with zero errors across all packages.
- [ ] `pnpm test` runs without error (may have zero test files at this stage).
- [ ] `pnpm gen:contract` completes without error and updates `packages/api-client/src/`.
- [ ] No `any` types in any committed TypeScript file.
- [ ] No `.env` files in git (verify with `git status`).
